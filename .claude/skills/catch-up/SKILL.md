# 세션 캐치업

새 세션을 시작할 때 프로젝트가 어디까지 왔는지 파악하고 요약한다.

## 절차

1. 다음을 읽는다 (없는 파일은 건너뛴다):
   - `docs/TODO.md` — 진행 중/대기 항목
   - `docs/DONE.md` — 최근 3개 항목만
   - `git log --oneline -10` 과 `git status --short`
2. 커밋되지 않은 변경이 있으면 `git diff --stat`으로 어떤 작업의 흔적인지 파악한다.

## 출력 형식

아래 3개 섹션으로 간결하게 요약한다. 파일 내용을 그대로 나열하지 말고 종합해서 쓴다.

**지금까지**: 최근 완료된 작업과 거기서 내린 주요 결정 (DONE.md 기반, 2~3줄)

**현재 상태**: 진행 중인 작업과 워킹 트리 상태. 커밋 안 된 변경이 TODO의 어느 항목에 해당하는지 연결해서 설명.

**다음 할 일**: TODO 기준으로 가장 우선인 작업 1~2개와, 바로 시작한다면 첫 단계가 뭔지 제안.

## 규칙

- 요약만 하고 멈춘다. 사용자가 시키기 전에 작업을 시작하지 않는다.
- TODO.md와 실제 코드 상태가 어긋나 보이면 (예: 완료된 것 같은데 TODO에 남아있음) 그 사실을 지적한다.
