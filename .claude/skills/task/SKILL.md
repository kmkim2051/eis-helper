# 작업 컨텍스트 관리

`docs/task/TODO.md`(할 일)와 `docs/task/DONE.md`(완료 기록) 두 파일을 관리한다.
파일이 없으면 아래 형식으로 새로 만든다.

## 사용법

- `/task` (인자 없음) — TODO.md의 현재 목록을 우선순위 순으로 보여준다.
- `/task add <내용>` — TODO.md에 항목을 추가한다. 내용이 모호하면 한 줄로 다듬어서 넣는다.
- `/task done <내용 또는 번호>` — 해당 항목을 TODO.md에서 지우고 DONE.md 맨 아래에 append한다.

## TODO.md 형식

```markdown
# TODO

## 진행 중
- [ ] 항목 (착수일: YYYY-MM-DD)

## 대기
- [ ] 항목
```

- 항목은 한 줄로, 완료 조건이 판단 가능한 수준으로 구체적으로 쓴다.
- "진행 중"은 동시에 1~2개만 유지한다. 넘치면 사용자에게 확인 후 "대기"로 내린다.

## DONE.md 형식 (작성 시간 오름차순 — 새 기록은 파일 맨 아래에 append)

```markdown
# DONE

## YYYY-MM-DD 작업 제목
- 변경: 주요 변경 파일/패키지
- 결정: 이 작업에서 내린 설계 결정과 이유 (한두 줄)
- 남은 것: 후속으로 미룬 일 (있으면 TODO.md에도 추가)
```

## 규칙

- `done` 처리 시 반드시 **결정 사항**을 채운다. 대화에서 파악한 내용으로 초안을 쓰고, 파악이 안 되면 사용자에게 짧게 물어본다. 이 기록이 다음 세션의 컨텍스트가 된다.
- "남은 것"에 적은 항목은 TODO.md에도 자동으로 추가한다.
- 커밋과 무관하게 파일만 수정한다. 커밋은 사용자가 요청할 때만.
